Fallos de Transacción

¿Qué es? Son errores que ocurren cuando una transacción no logra completarse correctamente, ya sea por conflictos de concurrencia, errores lógicos o interrupciones inesperadas.

¿Cómo se maneja? Se utilizan mecanismos como COMMIT y ROLLBACK. Si la transacción se completa exitosamente, se confirma con COMMIT; si ocurre un fallo, se revierte con ROLLBACK para mantener la consistencia.

Ejemplo real: En una aplicación bancaria, un usuario transfiere dinero de una cuenta a otra. Si el sistema descuenta el monto de la primera cuenta pero falla antes de acreditar la segunda, la transacción se revierte.

BEGIN TRY
   BEGIN TRANSACTION;
   UPDATE Cuentas SET Saldo = Saldo - 500 WHERE IdCuenta = 1;
   UPDATE Cuentas SET Saldo = Saldo + 500 WHERE IdCuenta = 2;
   COMMIT TRANSACTION;
END TRY
BEGIN CATCH
   ROLLBACK TRANSACTION;
   PRINT 'Error en la transacción. Se ha revertido.';
END CATCH;

Fallos de Sistema

¿Qué es? Se producen cuando el sistema operativo o el servidor de base de datos se interrumpe inesperadamente, por ejemplo, por un corte de energía o un error del software.

¿Cómo se maneja? Las bases de datos modernas emplean logs de transacciones para recuperar el estado anterior al fallo. Al reiniciar, el sistema revisa el log y restaura la base de datos al último estado válido.

Ejemplo real: Un supermercado actualiza su inventario y ocurre un apagón. Al reiniciar, el sistema usa los registros para restaurar el inventario.

SELECT name, recovery_model_desc
FROM sys.databases;

Fallos de Medio

¿Qué es? Son fallos físicos en el hardware, como daños en el disco duro o corrupción de archivos.

¿Cómo se maneja? Se requiere restaurar la base de datos desde una copia de seguridad y aplicar los logs de transacciones para recuperar la información perdida.

Ejemplo real: Una empresa sufre daños en el disco duro de su servidor. Gracias a respaldos diarios y logs, logra recuperar la información hasta el día anterior al fallo.

RESTORE DATABASE MiBase
FROM DISK = 'C:\\Backups\\MiBase.bak'
WITH RECOVERY;

Errores de Usuario/Operación

¿Qué es? Son errores provocados por acciones humanas, como borrar datos accidentalmente o ejecutar consultas incorrectas.

¿Cómo se maneja? Se previenen con permisos adecuados, validaciones en las aplicaciones y respaldos frecuentes. Algunas bases de datos permiten recuperar datos eliminados recientemente.

Ejemplo real: Un empleado elimina por error la tabla de clientes. Gracias a un respaldo automático, la empresa restaura la información sin mayores pérdidas.

RESTORE DATABASE MiBase
FROM DISK = 'C:\\Backups\\MiBase.bak'
WITH RECOVERY;

Clasificación Crítica de Fallos

Curso: Base de Datos 2 | Catedrático: Carlos Godínez

En SQL Server, la gestión de fallos no es opcional, es fundamental para la supervivencia del negocio. Analizar los fallos según su origen y alcance nos permite diseñar estrategias de recuperación robustas (DRP) que garanticen la continuidad operativa.

🔄 Fallos de Transacción

Origen y Alcance: Estos fallos se originan a nivel lógico, dentro del flujo de la unidad de trabajo. Su alcance suele limitarse a la transacción en ejecución, sin afectar la integridad física de los datos si el motor maneja correctamente el rollback.

Ejemplo Real: En una transferencia bancaria, el sistema descuenta el saldo de la Cuenta A pero el servidor falla antes de acreditar a la Cuenta B. El rollback automático asegura que el dinero se reintegre a A instantáneamente.

BEGIN TRY
   BEGIN TRANSACTION;
   -- restamos el saldo aki
   UPDATE Cuentas SET Saldo = Saldo - 500 WHERE IdCuenta = 1;
   UPDATE Cuentas SET Saldo = Saldo + 500 WHERE IdCuenta = 2; -- si esto falla
   COMMIT TRANSACTION;
END TRY
BEGIN CATCH
   -- clavo, regresamos todo como estaba
   ROLLBACK TRANSACTION;
   PRINT 'Error. Rollback para q no se pierda el dato';
END CATCH;

💻 Fallos de Sistema

Origen y Alcance: Se originan por fallas de hardware externo (apagones) o errores del sistema operativo. Su alcance es mayor, pues afecta a todas las transacciones que estaban activas en la memoria volátil (RAM) en el momento del "crash".

Ejemplo Real: Un servidor de inventario sufre un apagón repentino. Al reiniciar, SQL Server usa el Log de Transacciones para asegurar que las ventas confirmadas persistan (Redo) y las incompletas se reviertan (Undo).

-- comando para ver q modelo d recuperacion tiene la bd
-- para saber q tan protegidos estamos
SELECT name, recovery_model_desc FROM sys.databases;

💽 Fallos de Medio

Origen y Alcance: Tienen su origen en fallas físicas del hardware de almacenamiento (discos duros corruptos, RAID fallido). Su alcance es crítico, pues puede dañar o eliminar definitivamente los archivos .mdf y .ldf.

Ejemplo Real: El disco sólido de un servidor contable falla y deja de leerse. La recuperación requiere montar un disco nuevo y restaurar el último Backup Completo y los Backups d Log secuenciales.

-- restaurando la bd x q el disco se arruino
RESTORE DATABASE MiBase FROM DISK = 'C:\\Backups\\MiBase.bak' WITH NORECOVERY;
-- aki aplicamos los logs para q no se pierda info
RESTORE LOG MiBase FROM DISK = 'C:\\Backups\\LogActual.trn' WITH RECOVERY;

👤 Errores de Usuario

Origen y Alcance: Originados por la negligencia o accidentes humanos (borrados accidentales, consultas mal formadas). Su alcance es catastrófico, ya que el motor ejecuta órdenes válidas que destruyen datos en segundos.

Ejemplo Real: Un DBA junior ejecuta un `DELETE FROM Clientes` sin cláusula `WHERE` en producción, borrando millones d registros. La única solución es usar los respaldos de log para volver al estado de la BD un minuto antes del error.

-- toca recuperar el respaldo q se hizo antes d q borraran todo x error
RESTORE DATABASE MiBase FROM DISK = 'C:\\Backups\\RespaldoSeguro.bak' WITH RECOVERY;

⚙️ Mecanismos de Recuperación

Write-Ahead Logging (WAL): SQL Server garantiza que toda modificación quede primero en el Log de Transacciones antes de escribirse en disco. Esto permite reconstruir el estado exacto de la BD ante cualquier fallo inesperado.

Checkpoints: El motor realiza puntos de control periódicos que "vacían" las páginas sucias de la memoria al disco, reduciendo el tiempo de recuperación tras un crash. Sin checkpoints, el proceso de Redo podría tomar horas.

Algoritmo ARIES: Es el algoritmo base de SQL Server. Opera en tres fases: Análisis (identifica transacciones activas al momento del fallo), Redo (reaplica todos los cambios confirmados) y Undo (revierte las transacciones incompletas).

-- Ver el espacio usado por el log de transacciones
DBCC SQLPERF(LOGSPACE);

-- Forzar un checkpoint manual
CHECKPOINT;

-- Ver transacciones activas actualmente
SELECT * FROM sys.dm_tran_active_transactions;
